## Теоретическая основа

Индекс уязвимости побережья (Coastal Vulnerability Index, CVI) был предложен Gornitz (1990) и впоследствии доработан Thieler & Hammar-Klose (1999, USGS). Он объединяет шесть физических переменных в единый индекс, отражающий **относительную уязвимость** береговых сегментов к подъёму уровня моря:

$$
\text{CVI} = \sqrt{\frac{a \cdot b \cdot c \cdot d \cdot e \cdot f}{6}}
$$

где *a*–*f* — ранжированные значения (от 1 до 5) для каждой переменной. Результат представляет собой скрининговый индикатор, используемый для приоритизации мониторинга, территориального планирования и мероприятий по защите берегов.

CVI относится к индексным методам оценки: он не предсказывает конкретные деформации берега, а ранжирует участки по степени уязвимости от «очень низкой» до «очень высокой».

## Основные возможности

- **Автоматический расчёт CVI** по классической формуле Gornitz/USGS (квадратный корень из среднего геометрического)
- **Геопространственный ввод/вывод** — чтение и запись Shapefile, GeoPackage, GeoJSON через GeoPandas
- **Автоматический поиск и агрегацию отсутствующих геопространственных данных** - поиск и получение данных из открытых источников
- **Встроенная реклассификация** непрерывных растровых данных (уклон ЦМР, высота волн) в ранговые категории
- **Трансектный анализ** — сегментация береговой линии на равноудалённые трансекты с расчётом индекса для каждого
- **Визуализация** — генерация классифицированных карт CVI (Matplotlib / Plotly) с экспортом в PNG/HTML
- **Расширяемость** — возможность добавления пользовательских переменных или альтернативных формул индекса (ICVI, PCVI)

```python
from src.config import load_config
from src.preprocessing import prepare_layers
from src.cvi import compute_cvi
```

## Входные данные

Калькулятор использует шесть стандартных переменных CVI. Каждая может быть передана как атрибут векторного слоя или растровый слой:

| Переменная | Описание | Типичный источник | Единицы |
|------------|----------|-------------------|---------|
| **Геоморфология** | Тип берегового рельефа (скалистый клиф, песчаный пляж, эстуарий и др.) | Ручная классификация / карты наземного покрова | Категориальная (1–5) |
| **Скорость изменения береговой линии** | Историческая скорость эрозии / аккреции | Анализ DSAS, CoastSat, спутниковые снимки | м/год |
| **Уклон прибрежной зоны** | Региональный градиент берега | ЦМР (SRTM, FABDEM, LiDAR) | % или градусы |
| **Скорость подъёма уровня моря** | Долгосрочный тренд SLR | Мареографы PSMSL, спутниковая альтиметрия | мм/год |
| **Средняя значительная высота волн** | Прокси энергии волн | Реанализ ERA5, модель SWAN | м |
| **Средний приливной диапазон** | Амплитуда приливов | FES2014, данные мареографов | м |

> **Совет для российских побережий:** используйте мареографы Росгидромета, FABDEM для расчёта уклона и реанализ ERA5 для параметров волнения. Пошаговый пример для Балтийского моря — в [Jupyter-ноутбуке](notebooks/example_workflow.ipynb).

## Конфигурация

Все пороги ранжирования и параметры обработки задаются в файле `config/default_config.yaml`:

```yaml
study_area:
  name: "Самбийский полуостров"
  crs: "EPSG:32634"              # UTM зона 34N
  coastline: "data/raw/coastline.shp"

transects:
  spacing_m: 500                  # Расстояние между трансектами (м)
  length_m: 2000                  # Длина трансекта — в сторону суши + моря (м)

ranking:
  shoreline_change_rate:          # м/год → ранг
    1: [">2.0"]                   # Очень низкая уязвимость (аккреция)
    2: ["1.0 — 2.0"]
    3: ["-1.0 — 1.0"]            # Умеренная
    4: ["-2.0 — -1.0"]
    5: ["< -2.0"]                # Очень высокая (сильная эрозия)

  coastal_slope:                  # % → ранг
    1: [">1.2"]
    2: ["0.9 — 1.2"]
    3: ["0.6 — 0.9"]
    4: ["0.3 — 0.6"]
    5: ["<0.3"]

  sea_level_rise:                 # мм/год → ранг
    1: ["<1.8"]
    2: ["1.8 — 2.5"]
    3: ["2.5 — 3.0"]
    4: ["3.0 — 3.4"]
    5: [">3.4"]

  wave_height:                    # м → ранг
    1: ["<0.55"]
    2: ["0.55 — 0.85"]
    3: ["0.85 — 1.05"]
    4: ["1.05 — 1.25"]
    5: [">1.25"]

  tidal_range:                    # м → ранг
    1: [">6.0"]                   # Макроприливный (низкая уязвимость)
    2: ["4.0 — 6.0"]
    3: ["2.0 — 4.0"]
    4: ["1.0 — 2.0"]
    5: ["<1.0"]                   # Микроприливный (высокая уязвимость)

  geomorphology:                  # Категориальное соответствие
    1: ["Скалистый клиф"]
    2: ["Средний клиф"]
    3: ["Низкий клиф", "Аллювиальная равнина"]
    4: ["Галечный пляж", "Эстуарий"]
    5: ["Песчаный пляж", "Ватты", "Барьерный берег"]

output:
  format: "gpkg"                  # gpkg | shp | geojson
  cvi_classes:
    low:       [0, 7]             # Низкая уязвимость
    moderate:  [7, 15]            # Умеренная
    high:      [15, 25]           # Высокая
    very_high: [25, null]         # Очень высокая
```

## Результаты

Пайплайн генерирует следующие выходные данные:

- **CVI GeoPackage / Shapefile** — геометрия трансектов с ранговыми значениями каждой переменной и итоговым CVI
- **Карта CVI (PNG / HTML)** — классифицированная карта с категориями уязвимости

## Методология

1. **Сегментация береговой линии** — входная береговая линия делится на равноудалённые трансекты
2. **Извлечение данных** — значение каждой переменной присваивается соответствующему трансекту
3. **Ранжирование** — непрерывные значения реклассифицируются в порядковые ранги (1 = низкая уязвимость, 5 = высокая уязвимость) на основе настраиваемых пороговых значений
4. **Расчёт индекса** — CVI вычисляется по формуле:

$$
\text{CVI} = \sqrt{\frac{\prod_{i=1}^{n} x_i}{n}}
$$

где $x_i$ — ранговые значения переменных, $n$ — количество переменных.

5. **Классификация** — значения CVI группируются в категории уязвимости (низкая, умеренная, высокая, очень высокая) на основе перцентильных или фиксированных порогов
6. **Экспорт** — результаты сохраняются в виде геореференцированных векторных данных и статических/интерактивных карт

**Минимальный пример на Python:**

```python
import geopandas as gpd
from src.cvi import compute_cvi_from_geodataframe

# Загрузка предварительно ранжированных трансектов
gdf = gpd.read_file("data/processed/ranked_transects.gpkg")
```
